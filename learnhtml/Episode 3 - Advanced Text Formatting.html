Use <Img> to insert images in HTML document or webpage.

Use Absolute File Path for src attribute

Absolute file path is the exact location of the image on the webs. It start with the domain uri and path the image file.
e.g: http://domain.com/path-to-image/image-name.jpg

Example

<img src="http://127.0.0.1:5500/images/image-720.jpg" alt="">

Insert External Image into HTML Document

Also, you can insert the external images from others' websites into your webpage using <img> tag. Mostly, it the purpose of using absolute file path.

e.g: http://greatisprogramming.com/wp-content/uploads/2020/02/HTML-1024x576.jpg
Thanks, greatisprogramming.com for providing the image for following example

Example:


        

            <p style="display: block; width: 380px;">

                <img 
                
                    src="http://greatisprogramming.com/wp-content/uploads/2020/02/HTML-1024x576.jpg"

                    alt="HTML5 Late Night Image Cover"

                    width="100%"

                    height="auto"

                >

            </p>

As displayed in the above code block, we use src attribute and absolute path to the external image from greatisprogramming.com
also, we used the <img> with additional three attributes:

  • Alternative attribute (atl) to display the aternative property if the property of src attribute is missing or incorrect
  • width attribute to set the with of the current image to 100% of parent container(<p> tag)
  • height attribute to set proper asect radio of the image dimention

Use Relative for Image in HTML Document

Absolute file path is an accurate way to tell the HTML parser or browser regarding the correct locatoin of an image.
However, we will find a bit difficult to manage images and resources in the HTML document using absolute.
Think about change of your domain name, or move project from local host to live server.
You will have to make sure you put the correct domain uri in every single page of your HTML documents.

It's the reason why the relative paths come up. You just tell the HTML parser or browser where to look for the image
by related the image location to file base directory or root base directory

For example,

<img src="images/image-720.jpg">

The above code tells the HTML parser or browser to look for image-720.jpg in images directory where
the current HTLM document (Episode 4 - Working with Media Contents.html) based in (root directory)

Use Multiple Image Sources for Responsive Image

Providing the correct image dimention and file size helps with improve better user experiences.
It can save users' cost of downloading the large image file or provide the high quality image in big screen devices.
What's more, it makes the website load faster as browser choose the correct image to be display.

To tell the browsers or HTML parser to use multiple image sources, you can use srcset attribute.
Its property is the combination of image sources and descrptors separated by ( , ).

For Instance:


        

            <img
                src="images/image-720.jpg"
                srcset="/images/image-720.jpg 720w,
                        /images/image-480.jpg 480w,
                        /images/image-375.jpg 375w"
            >

The above code tells browsers that the image has multiple sources with different dimentions.
The w behind the descriptors stand for pixel width

We told the browsers we have multiple image soruces, yet we didn't the browser when to use the sources. You have to specify that in the sizes attribute.

The sizes attribute's property contains the combination of media query and pixel width.

For example:

        
            <img

                src="images/image-720.jpg"

                srcset="/images/image-720.jpg 720w,
                        /images/image-480.jpg 480w,
                        /images/image-375.jpg 375w"

                sizes="(min-width: 1200px) 720px,
                (min-width: 768px) 480px,
                (min-width: 375px) 375px,
                100vw"
            >

The above code tells the HTML parser to do the follow works:

  1. Display 720 pixel width image when users access from any device that screen is larger than or equal to 1200 pixel width
  2. Display 480 pixel width image when users access from any device that screen is larger than or equal to 768 pixel width
  3. Display 375 pixel width image when users access from any device that screen is larger than or equal to 375 pixel width
  4. Display image on available screen when access from any divice that screen is smaller than 375 pixel width
